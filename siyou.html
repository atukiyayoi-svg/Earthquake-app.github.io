EPICA 仕様書及び取扱説明書

バージョン: V4.4.2 R.ver

1. システム概要とアーキテクチャ (Detailed Architecture)

本システム「EPICA」は、各種外部APIから取得したデータを基に、リアルタイムの地震動（強震モニタ）、緊急地震速報（EEW）、津波情報、各種気象情報を2Dマップ(Leaflet)上で総合的に監視・可視化・自動読み上げを行う統合防災ダッシュボードです。
大元の観測機関（気象庁・防災科研）のデータが、どのように本システムに統合されているかを以下に示します。

2. 主要機能の解説と視覚的アルゴリズム

緊急地震速報 (EEW) と 高精度予測震度計算

EEW受信時、震源情報とP波・S波の想定到達円をCanvas上でリアルタイムにアニメーション描画します。予測震度は、以下のプロセスを経て算出・描画されます。

  1. ARVの取得: 各市区町村ポリゴンの中心座標を基に、J-SHIS APIから表層地盤増幅率(ARV)を取得。
  2. 距離減衰式の適用: 震源の深さ、マグニチュード、震源距離から工学的基盤の最大速度(PGV600)を計算し、ARVを乗算して地表の計測震度を予測します。
  3. 海域オフセット補正: 震央が海域の場合、気象庁発表の「最大予測震度」を下回る計算結果になった際、陸地との距離ギャップを埋めるためのオフセット値を動的に算出し加算します。

強震モニタ 揺れ検知クラスタリング (Shake Detection)

Yahoo! API経由で取得した観測点データ（毎秒更新）を監視します。単発のノイズによる誤検知を防ぐため、独自のクラスタリング処理を実装しています。

  • ユーザー設定閾値（デフォルト: 計測震度目安1.0以上）を超える観測点を抽出。
  • 超過した観測点から半径40km以内に別の超過観測点が存在し、かつ4点以上の観測点で有意な揺れを検知した場合に発報。
  • ガイガー法に基づいた最適化により、揺れの検知時間差から高精度な震央推定を行います。

過去データの記録と再現機能 (Reproduction)

有事の際、システムの動作状況をそのまま記録し、後から再現・検証することができる機能です。

  • EEWを受信すると、受信の10秒前に遡って自動的に記録（レコーディング）を開始します。
  • 記録データには、強震モニタのリアルタイムデータ、EEWの電文、気象庁APIの確定報、XML情報が含まれます。
  • 確定報を受信後、一定時間経過で自動的に記録を終了し、JSONファイルとしてエクスポートします。後日インポートすることで、当時のマップ描画や音声案内を完全再現できます。

年間地震活動アニメーション (Annual Animation)

USGS APIから今年発生した地震データを取得し、経過時間に合わせて描画する機能です。

  • 右サイドバーの専用マップ上で、年始めからの地震発生状況をマグニチュードに応じた波紋エフェクトで視覚的に再現します。
  • 地震活動の活発化や群発地震の推移を直感的に把握できます。

3. 情報パネルの完全解説

左サイドバー・メイン画面

  • 最新情報パネル: 最新の地震の最大震度、震源地、M、深さを表示。津波調査中の場合は「調査中」バッジが点滅します。
  • 震度詳細パネル: 各都道府県・市区町村の観測震度を高い順にスクロール表示。
  • 津波パネル: P2P APIと気象庁XMLの双方のデータを統合し、大津波警報・津波警報・注意報・予報のエリア、到達予想時刻、予想高さを表示。
  • EEWコンテナ: 複数のEEWを同時に受信した場合、カード形式でスタック表示。PLUM法や訓練報のタグも識別します。
  • マップスワップ機能: メインの地震・EEWマップと、気象レーダーマップをクリック一つで入れ替えることが可能です。

右サイドバー・各種速報

  • 気象レーダー (RAIN / THUNDER): 気象庁の高解像度降水ナウキャスト(hrpns)と雷ナウキャスト(thns)のタイルを表示。1分おきに自動で表示モードが切り替わります。
  • 天気予報ティッカー: 画面下部をスクロールするティッカーで、全国の今日・明日の天気、降水確率、最高/最低気温をリアルタイム表示します。
  • J-ALERT (国民保護情報): 弾道ミサイル情報や緊急警報を専用プロキシ経由で取得し、画面中央にポップアップ表示します。
  • 地震情報ウィンドウ: 過去30件の地震履歴。クリックでマップと左パネルに当時の状況を復元表示。震源要素が更新された場合は自動で上書き修正されます。
  • 気象庁防災情報XML (JMA XML) 総合: eqvol.xml 及び extra.xml を定期的にフェッチし解析。気象特別警報や南海トラフ地震臨時情報などをハイライトします。

4. 高度なカスタマイズと設定機能

画面中央下部の「⚙ 設定」ボタンから、UIや動作を詳細にカスタマイズできます。設定は localStorage に保存され、次回起動時に復元されます。

  • UI・外観設定:
    • ベースマップ(地図タイル)表示: 背景の地図タイルのON/OFF切り替え。
    • カラーカスタマイズ: アプリのベース背景色、海（マップ背景）の色、陸地のベースカラー（透過も可能）、および震度(1〜7)ごとのカラーコードを個別に指定可能。
    • パネル別フォントサイズ: 各コンポーネントの文字サイズを独立して80%〜150%の間で調整可能。
  • 動作拡張・API設定:
    • マップ自動切り替え: メインマップと気象マップを1分おきに自動でスワップする機能。
    • J-ALERT(国民保護情報)の受信ON/OFF設定。
    • プッシュ通知 (Notification API) の有効化。
    • EEWソース切替: 無料の「Wolfx」と、有償トークン接続による「AXIS API」をシームレスに切り替え可能。
  • 再現機能 (REPRODUCTION) 管理:
    • ローカルに保存された記録データの手動エクスポート、および過去データのインポート機能。記録の強制終了ボタンも搭載。
  • 音声・BGM管理:
    • 時刻連動BGM（朝・昼・夕・夜）。有事には自動でフェードアウトしてミュートされます。
    • Web Speech APIによる自動読み上げ速度・ピッチ・音量の調整。
    • 各種通知SEの音量を個別にスライダーで設定可能。

⚠️ 予測機能に関する重要・免責事項

本システム上で描画される以下の情報は、独自のアルゴリズムに基づく完全な「予測」ならびに「推定」であり、正確なものではありません。

  • P波・S波の伝播サークルの描画: 理論値に基づく簡易的な描画であり、実際の地震波の到達を保証するものではありません。
  • J-SHISモデルを用いた「予測震度マップ」: 計算上の予測であり、地盤状況や断層の破壊過程により実際の観測震度とは大きく異なる場合があります。
  • 予測長周期地震動階級 (LPGM): マグニチュードと深さから簡易算出された参考値です。

本システムの情報を用いて生じた、いかなる損害・トラブルについても作者は一切の責任を負いません。実際の避難行動や意思決定の際は、必ず気象庁や自治体が発表する公式の防災情報に従ってください。

⚖️ 法律関連の注意事項 (気象業務法について)

日本国内において、気象庁以外の者が独自の計算式を用いて「予測震度」を作成し、これを不特定多数の一般大衆に向けて配信・通知する行為は、気象業務法に基づく気象庁長官の「予報業務許可（地震動）」が必要となる場合があります。

本システムをローカル環境等で実行し、個人的に利用する範囲（自己完結型の処理）であれば問題ありませんが、本システムの画面をライブ配信したり、生成された予測震度マップを不特定多数へ二次配信したりする行為は、関係法令に抵触する恐れがあるため十分にご注意ください。

🌐 取得API・データソースについて

  • 気象庁 (JMA): 各種観測データ、EEW、防災情報XML、気象レーダータイル、天気予報API
  • 防災科学技術研究所 (NIED): 強震観測網データ、表層地盤増幅率(ARV)データ
  • USGS (アメリカ地質調査所): 年間地震アニメーション用データ
  • Yahoo! 防災速報: リアルタイム震度データ (NIEDデータの中継)
  • P2P地震情報 API v2: 地震履歴一覧、津波警報エリア情報
  • Wolfx API / AXIS API (Prioris): 緊急地震速報、気象・ニュース速報

データの利用に関しては各配信元のオープンデータ利用規約に従ってください。
